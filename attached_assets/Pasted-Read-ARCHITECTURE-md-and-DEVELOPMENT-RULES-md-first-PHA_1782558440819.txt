TASK 2 — BACKEND PHONE AUDIT ENDPOINT

Add protected endpoint:

GET /api/crm/phone/audit

Return leads with phone numbers that may need cleanup.

For each lead:
{
id,
name,
phone,
normalizedPhone,
issue,
canAutoFix
}

Rules:

* If phone is empty/null, ignore.
* If normalizedPhone === phone and phone starts with "+", clean.
* If normalizePhone(phone) differs from phone and starts with "+", canAutoFix = true.
* If cannot confidently normalize, canAutoFix = false.
* Include issues like:

  * "Needs E.164 formatting"
  * "Possible PH trunk format"
  * "Unrecognized phone format"

TASK 3 — BACKEND BULK NORMALIZE ENDPOINT

Add protected endpoint:

POST /api/crm/phone/normalize

Body:
{
leadIds: number[]
}

Behavior:

* For each lead ID:

  * load lead
  * skip if no phone
  * normalize phone
  * update only if normalized value is different and looks valid
* Return:
  {
  updated,
  skipped,
  errors,
  results: [
  { leadId, oldPhone, newPhone, status, reason }
  ]
  }

Do not update numbers that cannot be safely normalized.
