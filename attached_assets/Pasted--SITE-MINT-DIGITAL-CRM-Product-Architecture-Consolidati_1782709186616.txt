# OBJECTIVE

Before implementing additional functionality:

Perform a full architectural audit of the CRM.

Identify:

* duplicate pages
* duplicate components
* duplicate API usage
* duplicate business logic
* duplicate navigation
* duplicated UI patterns
* overlapping modules
* inconsistent naming
* modules that should be merged
* features that belong together
* missing relationships between systems

Do NOT refactor yet.

Produce the architectural plan first.

# IMPLEMENTATION STRATEGY

Do NOT build everything at once.

Instead:

1. Audit the platform.

2. Identify what already exists.

3. Identify what should be merged.

4. Identify what should become standalone modules.

5. Recommend a new navigation hierarchy.

6. Recommend the implementation order.

7. Break the platform into Major Product Milestones.

Only after the architecture has been approved should implementation begin.

---

# IMPORTANT CONSTRAINTS

Do not touch:

* Backend
* Database schema
* Twilio
* Campaign scheduler
* Locked modules

unless the audit explicitly concludes they require future work.

Do not rebuild existing working systems.

Prefer consolidation over duplication.

Prefer extension over replacement.

---

# FINAL REPORT

1. Existing modules inventory
