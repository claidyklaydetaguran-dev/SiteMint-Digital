TASK 2 — BACKEND SEND ENDPOINT

Add protected endpoint:

POST /api/crm/campaigns/:id/send

Behavior:

1. Load campaign.
2. Load recipients.
3. For each recipient:

   * skip if no email
   * skip if already sent
   * send personalized_subject and personalized_body to the lead email
   * use existing Resend logic / test mode behavior
   * update recipient status:

     * sent
     * failed
     * skipped
   * set sent_at for successful sends
4. Send one at a time.
5. Add a small delay or safe rate limit between sends if easy.
6. Return summary:
   {
   sent,
   failed,
   skipped,
   results: [
   { recipientId, leadId, email, status, error }
   ],
   testMode
   }

IMPORTANT:
If CRM_EMAIL_TEST_MODE is true, do not send externally but still simulate safely and mark status in a clear way according to existing convention.
Do not send to anyone unless admin manually confirms in UI.

TASK 3 — RESEND FAILED ENDPOINT

Add protected endpoint:

POST /api/crm/campaigns/:id/recipients/:recipientId/resend

Behavior:

* Resend only that one failed or skipped recipient.
* Use stored personalized subject/body.
* Update status and sent_at.
* Return result.
