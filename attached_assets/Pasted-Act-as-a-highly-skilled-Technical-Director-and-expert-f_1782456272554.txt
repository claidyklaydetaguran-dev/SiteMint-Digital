TASK 1 — BACKEND SINGLE CONVERSION ENDPOINT

Create a protected endpoint:

POST /api/crm/import-discovery/:id

Behavior:

1. Find the discovery submission by id.
2. If not found, return 404.
3. Check if a CRM lead already exists by:

   * discoverySubmissionId matching the submission id, OR
   * same email if email exists.
4. If an existing CRM lead is found, do not duplicate it.
5. Return the existing lead with:

   * imported: false
   * existing: true
   * leadId
   * message: "Submission is already connected to a CRM lead."
6. If no lead exists, create a new crm_leads record.

Map fields:

* contactName → name
* companyName → company
* email → email
* phone → phone
* serviceInterest → serviceInterest
* tags → tags
* internalNotes → notes
* recommendedPackage → packageType
* discovery submission id → discoverySubmissionId
* source = "Discovery Form"
* status = "New"
* discoveryFormStatus = "Completed"

Priority mapping:

* leadScore >= 8 → High
* leadScore >= 5 → Medium
* otherwise Low

After creating the lead:

* Create a crm_activities record:
  type: "lead_imported"
  title: "Imported from Discovery Portal"
  description: Include the submission company/name and service interest if available.
  leadId: new lead id

Return:
{
imported: true,
existing: false,
leadId,
message
}
