Inbox module (core ticketing workspace)
This is a four/five-pane resizable layout: a ticket-category rail (All, AI Processing, Open, Pending, Solved, each with counts), a ticket list pane, a conversation/thread pane, and a right-hand contextual Details pane. Each pane header has its own collapse toggle so agents can hide the list or details panel to focus on the conversation.
The ticket list supports two display modes toggled at the bottom: a card/preview view (avatar, contact name, subject preview, channel badge such as Support/Phone/LiveChat/Email, status pill, timestamp) and a dense table/list view with sortable columns (checkbox select, Subject, Status, Last Message with sort arrows, Agent, Note), plus pagination controls (item count, page number, page-size selector). Message bubbles differ by channel: text bubbles for email/chat, and for voice calls an embedded audio player (play button, scrubber, duration, download icon) plus an AI-generated call summary card with a "View transcript" link.

Visual and interaction design language
Overall the UI uses a clean, light, neutral palette: white/very light gray panel backgrounds, thin 1px dividers between panes, a deep navy/near-black for primary buttons and active nav states, and a small set of accent colors reserved for semantics — green for "online"/success/voice, red/orange-red for "open" ticket status dot, blue for "pending," purple/indigo as the brand accent (logo, trial banner, AI sparkle icon, active nav highlight), and a soft yellow tint specifically for internal notes. Status and channel tags render as small rounded pill badges with an icon plus label. Typography is a standard sans-serif system font, bold semibold headings, medium-weight body text, muted gray for secondary metadata (timestamps, counts). Empty states consistently use a centered light gray line-art icon, a bold "No data"/"Select an Agent" headline, and a muted helper sentence. Loading states use pulsing gray skeleton blocks that mirror the shape of the content about to load (list rows, avatar circles, message bubbles). Interactive elements favor dropdown menus anchored to trigger buttons, small popovers for search/filters, collapsible sections marked with a chevron, and icon-only buttons with tooltips on hover. The overall information density is high (compact row heights, small icons) which is typical of professional support-agent tooling optimized for speed over whitespace.
