── PHASE 1 — Generator library ──────────────────────────────────────
Create lib/aiCampaign.ts (or artifacts/web-agency/src/lib/aiCampaign.ts
if generation logic belongs client-adjacent — your call based on
where the existing OpenAI integration lives, check
lib/integrations-openai-ai-server first). It should expose functions to:
1. Generate a single campaign draft: given a persona (from
   campaignTaxonomy.ts), topic, objective, and toneProfile, produce a
   subject + body draft matching that persona's recommendedTone.
2. Generate a full sequence draft: given the same inputs plus a
   desired number of steps, produce an array of steps, each with
   dayOffset, channel (email/sms/call_prompt/task), subject/body (for
   email), intentLabel, and sensible spacing (don't front-load all
   steps on day 0-1).
Reuse campaignTaxonomy.ts's existing personas/topics/blueprints as
grounding input to the prompt sent to the model — do not invent a
parallel taxonomy. Keep prompts to the AI model tightly scoped (no
open-ended "write anything") and always return structured output
(subject/body/steps as data, not prose) so the UI can render it into
form fields directly.
Verification for this phase: call the function directly (script or
route) with 2-3 different persona/topic combos, confirm structured
output every time, confirm it never returns something requesting
auto-send/auto-enroll action language.

Verification: curl the route with a real admin token, confirm shape,
