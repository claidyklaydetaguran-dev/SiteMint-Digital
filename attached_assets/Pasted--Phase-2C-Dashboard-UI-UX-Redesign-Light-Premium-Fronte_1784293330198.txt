## 1. Design System (create `src/index.css` tokens + consistent usage)

**Personality:** premium, calm, trustworthy business SaaS. Think Stripe Dashboard / Linear-light / Notion-crisp — NOT a dev console. Generous whitespace, clear hierarchy, one accent color used with restraint.

**Tokens (Tailwind v4 CSS variables in index.css; use everywhere, no ad-hoc colors):**

- Background app: `#F8FAFC` (slate-50) · Surface/card: `#FFFFFF` · Border: `#E2E8F0` (slate-200), hover `#CBD5E1`
- Primary accent: indigo-600 `#4F46E5` (buttons, active nav, links, focus rings); indigo-50 tints for active/selected backgrounds
- Text: slate-900 headings · slate-600 body · slate-400 muted/meta
- Semantic: success emerald-600 · warning amber-500 · danger rose-600
- Tier colors (consistent EVERYWHERE — badges, filters, details): Hot rose · Warm amber · Cold sky · Needs Review yellow · Disqualified slate · Opted out slate-outline
- Radius: cards `rounded-xl` (12px), buttons/inputs `rounded-lg` (8px), badges `rounded-full`
- Shadows: cards `shadow-sm`, elevate to `shadow-md` on hover for clickable cards only
- Type scale: page title 20px/semibold · section 16px/semibold · body 14px · meta 12px · KPI number 28–32px/bold tabular-nums
- Spacing: 4px base; card padding 20–24px; page gutter 24–32px
- Motion: 150–200ms ease transitions on hover/active/accordion; respect `prefers-reduced-motion`
- Focus: visible indigo focus ring on ALL interactive elements (`focus-visible:ring-2 ring-indigo-500 ring-offset-2`)

**The four-state rule (every data view, no exceptions):** loading = skeletons matching final layout dimensions (`animate-pulse`); empty = icon + one-line headline + one-line helper + (when useful) a CTA; error = message + Retry button; success. No blank screens, no spinners-as-page.

## 2. Per-Page Redesign Spec

### AppLayout (sidebar + top bar)
- Keep 2B's grouped structure; polish: 232px sidebar, white background, subtle right border; brand row (logo mark + "SiteMint" wordmark + "AI Receptionist" subtitle in muted text).
- Nav item: icon + label, active = indigo-50 bg + indigo-700 text + 3px indigo left bar; hover = slate-50.
- Trial chip: refined card style (slate-50 bg, rounded-lg) with progress bar; turns amber when ≥80% used; "Upgrade" text-link inside. Hidden when paid — replaced by a small "Pro plan" badge row.
- NEW top bar (per page, not global chrome): page title + one-line description on the left; contextual primary action on the right where relevant.
- Mobile (<768px): sidebar collapses to an overlay drawer with hamburger in a slim top bar. All pages must be usable at 375px wide.

### Login
- Centered card on soft slate-50 with subtle gradient wash; brand mark; larger touch targets; error copy inline under fields; loading state on button; link to signup unchanged.

### Overview
- Header: "Good {morning/afternoon/evening}, {firm.name}" + date.
- KPI tiles (3, per 2B deviation): bigger numbers, small trend caption ("this week"), icon in tinted square; clickable → Conversations (with matching filter where applicable — Hot tile → Conversations pre-filtered to Hot via querystring or state).
- Recent conversations: card list rows with phone, tier badge, snippet of last inbound message if available in payload (it is NOT in the list payload — use relative time + status instead; do NOT add an endpoint), relative time; hover elevate; click → /conversations.
- Getting-started card: promoted visual treatment (indigo-50 border card with checklist: "Describe your business ✓/✗ · Set greeting ✓/✗ · Add qualifying questions ✓/✗"), each linking to /receptionist. Uses existing agent-config query only.

### Conversations (Inbox) — biggest quick-win surface
- Keep 3-pane master–detail; restyle all three panes with the token system.
- **QUICK WIN — Tier filter chips:** horizontal chip row above the list (All · Hot · Warm · Cold · Needs Review · Disqualified · Opted out) with counts; combinable with the existing status categories (rail) and search. Pure client-side filtering — tier is already in the payload.
- **QUICK WIN — "Why this tier":** in DetailsPanel, a "Lead details" card showing tier badge + `disqualifyReason` when present, with plain-English framing ("Marked {tier}. Reason: {disqualifyReason}"). Data already in the payload.
- List rows: cleaner cards — phone (formatted for readability), status dot + tier badge, relative time; unread/new visual weight is out of scope (no read-state exists).
- ThreadPanel: SMS bubbles refined (caller = white card w/ border, AI = indigo-600); date separators between days (client-side grouping); the "manual replies coming soon" notice restyled as a subtle info bar.
- Empty states per filter combination ("No Hot leads yet — they'll appear here the moment your AI qualifies one").
- Mobile: list → full-width; selecting opens thread as a stacked view with a back button.

### AI Receptionist (AgentConfig) — second quick-win surface
- Header card: status ("Live on SMS" emerald badge), firm's industry, and the Test tab (still disabled stub, restyled).
- **QUICK WIN — Industry templates:** a "Start from a template" section (only when greeting AND questions are empty, or behind a "Use a template" button otherwise) with 6 cards: Law Firm, Home Services, Real Estate, Medical/Dental Clinic, Salon & Spa, General Business. Selecting one pre-fills the form fields client-side (user reviews, edits, saves via the EXISTING PATCH — no new endpoints). Template content defined in a new `src/lib/agentTemplates.ts`:
  - Each template: `industry`, `greetingMessage` (friendly, ≤300 chars, mentions business name placeholder "{business}"— substitute firm.name on apply), 4–5 `qualifyingQuestions` (e.g., Law Firm: "What type of legal issue are you dealing with?", "When did the incident happen?", "Have you spoken with another attorney?", "What outcome are you hoping for?"; Home Services: service needed, urgency/timeline, property type, preferred appointment window, budget range; analogous quality for the rest).
- Form: two-column layout on desktop (fields left, live "preview" card right showing how the greeting SMS will look in a phone mockup frame — purely visual, client-rendered); qualifying questions as a re-orderable list (up/down buttons fine; no dnd dependency) with character counters matching server limits (500/1000/200×6 — mirror exactly).
- Save UX: sticky save bar appears when dirty ("Unsaved changes — Save / Discard"); success toast.

### Contacts (placeholder — restyle only)
- Honest, premium "coming soon" state with a preview illustration of what's coming; CTA → Conversations. No backend.

### Billing
- Restyle Plan/Usage tabs with the token system; plan card comparison (Trial vs Pro) with feature bullets; usage tab gets a simple bar visualization of N/M (client-side; no chart lib). Upgrade button states: normal / loading / the existing "not configured" error surfaced as a friendly inline notice ("Billing isn't live yet — contact us to upgrade") instead of a raw error.

### Settings
- Restyle the two remaining panels (Members coming-soon, Language) consistently; add an "Account" info card (firm name, email, member-since from session — read-only).

## 3. Accessibility & Responsive Bar (applies to every page)

- Keyboard: every interactive element tabbable in logical order; Enter/Space activates; filter chips are buttons, not divs.
- `aria-label` on all icon-only buttons; badges have text (never color-only meaning — tier badges always show the word).
- Contrast: all text ≥ 4.5:1 against its background (check the slate-400 metas on white — bump to slate-500 where needed).
- Breakpoints: 375px (must work), 768px (sidebar drawer), 1024px+ (full layout). No horizontal overflow anywhere.

## 4. Hard Constraints (unchanged discipline)

- ALL changes inside `artifacts/helpdesk/src` (+ helpdesk `index.css`/tailwind config if needed) + the two docs. ZERO api-server, ZERO web-agency, ZERO lib/, ZERO schema, ZERO new npm dependencies (shadcn/tailwind/lucide already present).
- No new API endpoints, no changes to fetch paths/query keys/polling intervals.
- Server validation limits mirrored exactly in UI (greeting 500, description 1000, ≤6 questions × 200).
- The wouter base path must keep working; all links relative as today.
