TASK 1 — Generate Steps from Blueprint

In CrmCampaignSequence.tsx, add:

“Generate Steps from Blueprint”

Behavior:

* Uses selected SiteMint blueprint from campaignTaxonomy.ts.
* Converts blueprint.exampleSteps into campaign steps.
* Creates draft steps through existing campaign step API.
* Does not send.
* Does not enroll.
* Does not schedule messages unless existing API does that by design.

Step mapping:

* dayOffset = example step day
* channel = example step channel
* subject = topic hook or “Draft: [Topic Title]”
* body = placeholder body:

  * Purpose
  * Recommended CTA
  * Behavioral signal tested
  * Notes for future AI copy generation
* sendTime = 09:00
* businessDaysOnly = true
* If metadata JSON is available on campaign_steps, store:

  * personaId
  * topicId
  * objective
  * desiredBehavior
  * targetSignal
  * expectedLift
  * routingHint

If no metadata field exists, store the metadata as a structured note inside body and document limitation.

Safety:

* If campaign already has steps, ask:

  * Append generated steps
  * Replace existing draft steps only if safe
  * Cancel
* Never delete sent/scheduled history.
* Show clear confirmation:
  “This creates draft sequence steps only. It will not send or enroll anyone.”

TASK 2 — Step Intelligence Editor

In each step editor/card, add a small collapsible section:

“Step Intelligence”

Fields:

* Objective
* Desired behavior
* Target signal
* Expected lift
* Routing hint

Examples:

* Objective: “Test CRM automation interest”
* Desired behavior: “Click case study”
* Target signal: “crm_interest”
* Expected lift: “+12 Client Intent”
* Routing hint: “If clicked, move to CRM/Automation campaign”

Save this data:

* Prefer existing JSON/metadata field if available.
* Otherwise append it to the step body under a clearly marked:
  “[Step Intelligence]”
  section.
