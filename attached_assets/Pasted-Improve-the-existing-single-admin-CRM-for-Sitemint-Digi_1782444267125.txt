https://sitemintdigital.com/admin/crm

/admin/crm/dashboard

PHASE 2 — DEALS PIPELINE / KANBAN BOARD

Create a Deals table linked to contacts.

Database table: deals

Fields:

* id UUID primary key
* contact_id UUID nullable or not null depending on current CRM structure
* name text not null
* value numeric default 0
* stage text not null default 'Lead'
* close_date date nullable
* created_at timestamp default now()
* updated_at timestamp default now()

Allowed stages:

* Lead
* Qualified
* Proposal
* Won
* Lost

Use a proper Drizzle migration.

Build a beautiful drag-and-drop Kanban board at:

/admin/crm/deals

Use either:

* @hello-pangea/dnd
  or
* simple HTML5 drag-and-drop if dependency issues happen.

Kanban Requirements:

* One column per stage
* Deal cards inside each stage
* Drag deal cards between stages
* Updating stage should persist to the database
* Each stage column should show:

  * total number of deals
  * total deal value
  * average deal value if easy to add
* Smooth UI interactions
* Loading states
* Empty states
* Error states
* Mobile responsive fallback

Deal Card Should Show:

* Deal name
* Linked contact name if available
* Deal value
* Close date
* Stage badge

API Routes Needed:

* GET /api/crm/deals
* POST /api/crm/deals
* PATCH /api/crm/deals/:id
* DELETE /api/crm/deals/:id
* PATCH /api/crm/deals/:id/stage

Make sure all routes are protected by admin authentication.

PHASE 3 — CONTACT ACTIVITY TIMELINE

Create an Activities table linked to contacts.

Database table: activities

Fields:

* id UUID primary key
* contact_id UUID not null references contacts(id) on delete cascade
* type text not null
* content text not null
* created_at timestamp default now()

Allowed activity types:

* Call
* Email
* Meeting
* Note
* Status Change

Build this into the Contact Detail page:

/admin/crm/contacts/:id

or use the existing lead/contact detail route if already present.

Activity Timeline Requirements:

* Chronological feed, newest first
* Activity icon based on type
* Timestamp
* Activity content
* Clean card-based timeline design
* Empty state if no activities exist

Quick Log Form:
At the top of the contact detail page, add a quick activity form with:

* Activity type dropdown
* Content textarea
* Submit button

Also allow quick status updates:

* Change contact/lead status
* When status changes, automatically create a “Status Change” activity entry

API Routes Needed:

* GET /api/crm/contacts/:id/activities
* POST /api/crm/contacts/:id/activities
* PATCH /api/crm/contacts/:id/status
