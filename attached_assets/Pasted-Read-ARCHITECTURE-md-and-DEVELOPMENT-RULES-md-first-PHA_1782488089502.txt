Add to crm_campaign_recipients:

Use Drizzle migration/push properly.

Optional if easy:
Add index on resend_email_id for webhook lookup.

TASK 3 — STORE RESEND EMAIL ID AT SEND TIME

During campaign send and single-recipient resend:

* Capture the email id returned by Resend.
* Store it in crm_campaign_recipients.resend_email_id.
* If CRM_EMAIL_TEST_MODE is true and no real Resend id exists, store null and do not create fake event IDs.
* Do not break existing test mode.

TASK 4 — ADD RESEND WEBHOOK ENDPOINT

Add public endpoint:

POST /api/crm/webhooks/resend

No admin auth guard.

Security:

* Use RESEND_WEBHOOK_SECRET.
* Verify the webhook signature if Resend/Svix verification is already supported or can be added safely.
* If the secret is missing, return a clear 500/disabled response in non-production or reject safely.
* Do not accept unverified production webhooks.

Parse event types:

* email.opened
* email.clicked
* email.bounced
* email.delivery_failed

For each event:

1. Extract Resend email id from payload.
2. Find crm_campaign_recipients row by resend_email_id.
3. If no match, return 200 with ignored: true.
4. If matched, insert crm_campaign_events:

   * campaign_recipient_id
   * event_type:

     * opened
     * clicked
     * bounced
     * failed
   * metadata
   * occurred_at from payload timestamp if available, otherwise now()
5. For bounce or delivery failure:

   * update recipient status to failed
   * update last_error with event reason if available

Important:

* Webhook must be idempotent if possible.
* If duplicate events come in, do not crash.
* Do not modify Twilio webhook routes.
